# Code Quality Reference

## Default Skeleton

```text
src/data/
src/models/
src/training/
src/evaluation/
src/metrics/
src/utils/
configs/experiment/
configs/smoke/
scripts/figures/
tests/
```

Generate when useful:

```bash
python scripts/render_project_skeleton.py --project .
```

## Contract Check

```bash
python scripts/check_experiment_contract.py \
  --experiment-id EXP-001 \
  --config configs/experiment/EXP-001.yaml \
  --smoke-config configs/smoke/EXP-001-smoke.yaml \
  --registry docs/thesis/experiment-registry.md
```

Add `--require-outputs` after a run to require `manifest.json`, `config_resolved.json`, `metrics.json`, and `logs/`. Add `--require-env-snapshot --require-remote-artifact` for formal `remote_desktop_4060` or cloud GPU runs.

```bash
python scripts/write_environment_snapshot.py \
  --out outputs/EXP-001/environment.txt \
  --label remote_desktop_4060
```

## Remote 4060 Templates

Copy and customize:

```text
scripts/remote_sync_to_4060.sh.template
scripts/remote_run_4060.sh.template
scripts/remote_fetch_results.sh.template
scripts/remote_archive_experiment.sh.template
```

Only store SSH alias, paths, and commands. Never store passwords. The run template writes an environment snapshot before the formal command by default. The fetch template defaults to a lightweight local index; the archive template records the remote run folder and checksum path that should be copied into `experiment-registry.md`.

## Review Checklist

| Risk | Check |
|---|---|
| Data leakage | split creation and split use are separated and recorded |
| Config drift | comparable experiments share split, seed policy, and budget |
| Hard-coded state | paths, metrics, model names, and seed are config-driven |
| Silent failure | scripts fail loudly on missing inputs |
| Non-reproducible result | output directory contains machine-readable artifacts |
| Notebook-only figure | final figure/table can be regenerated by script |
