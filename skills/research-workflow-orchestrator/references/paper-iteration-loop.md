# Paper Iteration Loop

Research papers are iterative. New results can change the story, and audit findings can require new experiments or revised figures.

## Loop

```text
Project setup
  -> literature and gap
  -> experiment design
  -> experiment record
  -> result analysis
  -> claim-evidence map
  -> figures and tables
  -> section drafting
  -> final audit
  -> revise plan/results/figures/writing as needed
```

## Decision Points

- If the contribution is unclear, return to `$research-paper-plan`.
- If related work is weak, return to `$research-literature-review` and `$semanticscholar-skill`.
- If a claim lacks evidence, return to `$research-results-analysis` or experiment design.
- If a result is important but hard to see, return to `$research-paper-figures`.
- If a section is vague or overclaims, return to `$research-paper-writing`.
- If submission artifacts are inconsistent, return to `$research-final-audit`, `$doc`, `$pdf`, or LaTeX compilation.

## Done Criteria

A stage is done only when its output is traceable:

- Planning produces a claim/evidence map.
- Literature review produces a matrix with verified metadata.
- Experiments produce result artifacts and notes.
- Analysis produces supported and unsupported claim lists.
- Figures map to data and claims.
- Writing marks missing evidence and citations.
- Final audit gives prioritized fixes.
