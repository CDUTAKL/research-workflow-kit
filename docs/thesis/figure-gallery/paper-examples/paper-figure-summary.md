# Paper Figure Summary — 跨案例设计模式总结

> 从 paper-figure-notes.md 的 8 个案例中抽取的共性规律。
> 这些规律应作为设计新图时的第一参考，而非凭感觉画图。

---

## 一、高水平图表的共性特征

### 1. 信息层级 = 三段式

几乎所有论文的主图都遵循：

```
Panel a: 整体概览（schematic / flow / overview）
Panel b: 核心结果（bar / scatter / line — 最强的 claim）
Panel c/d: 补充验证 / 消融分析 / 效率对比
```

| 案例 | a (概览) | b (核心) | c/d (支撑) |
|------|---------|---------|-----------|
| AlphaFold2 | 数据流 overview | 核心架构 | 精修细节 |
| InstructGPT | 训练流程 | win rate bar | scaling + KL 消融 |
| scVI | 概率图模型 | latent space | 下游任务评估 |
| UMAP | N/A | MNIST scatter | 更多数据 + runtime |

**规则**：读者应该只看 panel a 就知道"这篇论文做了什么"，只看 panel b 就知道"核心结果是什么"。

### 2. 颜色编码 = 3-5 种

| 颜色数 | 出现频率 | 典型用途 |
|--------|---------|---------|
| 3 种 | ★★★ | 自己方法(1) + baseline(1) + 辅助(1) |
| 4 种 | ★★☆ | 上述 + 额外类别区分 |
| 5 种 | ★☆☆ | 复杂多方法对比 |

**核心策略**：
- 自己的方法 = 暖色/高饱和（红/橙/蓝）→ 视觉上"弹出"
- baseline 方法 = 灰色系 → 视觉上"后退"
- 这个策略在 scGPT、InstructGPT、UMAP 中一致使用

### 3. 图例处理 = 定义一次，别处继承

| 策略 | 占比 | 说明 |
|------|------|------|
| 第一个 panel 放图例，后续省略 | 60% | UMAP, scGPT 等 |
| 直接标注在图中 | 30% | AlphaFold2, Swin Transformer, Transformer |
| 每个 panel 独立图例 | 10% | 仅当颜色编码不同 |

### 4. Caption 写法 = 描述 + 不解释

```
标准结构：
第 1 句：概述整张图展示什么
第 2-N 句：a, ... b, ... c, ...（逐 panel 描述）
最后一句：(可选) 缩略词解释

反模式（应避免）：
- "我们的方法显著优于..." ← 这是正文的活，不是 caption
- "这说明..." ← caption 不解释"为什么"
- "值得注意的是..." ← 观点性语言不应出现在 caption
```

### 5. 定量 + 定性组合

| 组合模式 | 出现频率 | 示例 |
|---------|---------|------|
| scatter (定性) + bar (定量) | ★★★ | UMAP, scVI, scGPT |
| flow (方法) + bar (结果) + line (消融) | ★★☆ | InstructGPT |
| 单一 bar chart | ★☆☆ | 仅当 claim 足够单一时 |

## 二、常见错误和避免方法

| 错误 | 严重程度 | 避免方法 |
|------|---------|---------|
| 颜色超过 6 种 | 高 | 合并为 "自己(1) + baseline(3-4 代表性) + 其他(灰)" |
| Panel 间颜色编码不一致 | 高 | 首个 panel 定义颜色，后续 panel 严格继承 |
| Caption 过度解释 | 中 | caption 只描述"是什么"，"为什么"留给正文 |
| 字太小（< 6pt 在最终尺寸） | 高 | 用 figure-audit-standard.md 检查 |
| 图例遮挡数据 | 中 | 图例放图外（右侧/底部）或第一个 panel 角落 |
| bar chart 无数值标注 | 中 | 在 bar 顶部标数值，读者不应靠眼睛估算 |
| 黑白打印不可读 | 高 | 同时检查灰阶版本，用线型/纹理辅助区分 |
| 散点过多导致 overplotting | 中 | 调低透明度、减小点大小、或用 hexbin |
| 坐标轴标签不完整 | 中 | 每个轴必须有标签 + 单位 |
| log scale 不明确 | 高 | 用 "log₁₀" 标注，而非仅靠刻度暗示 |

## 三、设计决策速查表

当你不确定某个设计选择时，按以下优先级：

| 问题 | 推荐决策 | 反例 |
|------|---------|------|
| 用几种颜色？ | 3-4 种 | >6 种 |
| 自己的方法用什么颜色？ | 暖色（红/橙） | 灰色、浅蓝 |
| 图例放哪？ | 首 panel 内 / 图外右侧 | 覆盖在数据上 |
| bar 上标数值？ | 是 | 否 |
| 坐标轴从 0 开始？ | bar chart: 是; line/scatter: 否 | 一律从 0 / 一律不从 0 |
| 用 error bar？ | 有重复实验: 是; 单次: 否 | 无方差却加 error bar |
| 几个 panel？ | 3-5 个 | >7 个（除非是 supplemental） |
| 黑白打印检查？ | 每次必做 | 不做 |
| caption 多长？ | 3-6 句 | 单句太短 / 段落太长 |

## 四、按图型的推荐使用场景

| 图型 | 最佳场景 | 避免场景 |
|------|---------|---------|
| grouped_bar | 多方法 × 少指标 对比 | 方法>8 个时用横向 bar |
| line_trend | 连续变量变化趋势 | 离散分类比较 |
| heatmap | 二维矩阵数据 | 一维数据（用 bar 更好） |
| scatter | 降维可视化 / 相关性 | 分类对比 |
| multi_panel | 复杂 story 需要多视角 | 简单发现（一个 bar 就够） |
| log_bar | 跨数量级数据 | 相近量级（log 会削弱差异感） |
| confusion_heatmap | 分类结果 | 回归结果 |
| forest | 效应量 + 置信区间 | 单点估计 |
| ablation_barh | 消融实验 | 方法对比 |
| image_plate | 图像样本 | 非图像数据 |
