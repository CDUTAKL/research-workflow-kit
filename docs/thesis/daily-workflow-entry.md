# Daily Workflow Entry

## Purpose

Use this file as the lightweight daily entry point for the research workflow. The web dashboard can update it through the local-only control service, while `docs/thesis/` remains the source of truth.

## Update Rules

- Record only the current day's working state: stage, focus, goal, blocker, next action, completed note, and tomorrow handoff.
- Use this file to reduce daily cognitive load; keep detailed evidence in the specific source records such as `claim-evidence-map.md`, `section-citation-map.md`, `experiment-registry.md`, or `final-artifact-manifest.md`.
- Every dashboard write should also append `workflow-edit-log.md`.
